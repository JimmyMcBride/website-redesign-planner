# Repo Scan 2026-05-10

## Summary

This repository is the source for the `website-redesign-planner` skill. It combines workflow instructions, support references, and a bundled SvelteKit starter app that downstream projects copy and customize.

## Durable Findings

- Root-level logic is documentation-driven. `SKILL.md` is the operational contract; `README.md` is the human-facing install and usage guide.
- `references/research-data-rules.md` governs evidence quality and claim safety for competitor research, review data, SEO observations, and report recommendations.
- `references/sveltekit-tailwind-typescript.md` governs how the generated report app should be scaffolded, merged, structured, and verified.
- `assets/sveltekit-report-app-template/` is the main implementation payload of the skill. It is a Bun/SvelteKit/TypeScript/Tailwind starter app using static prerendering.
- The template app’s route contract is stable and deliberate:
  - `/` analysis
  - `/audit` source-site audit plus Lighthouse section
  - `/design-systems` three design-system directions
  - `/examples` index and four detail demos
- The planner workflow also emits a project-local design-system skill from `assets/project-design-system-skill-template.md`, so template/design changes may require updates to both the report app and the generated-skill template.

## Current Technical Notes

- The template app currently uses Bun-first scripts in `package.json`.
- The template app includes Lighthouse capture support via `scripts/capture-lighthouse.ts` and typed summary loading via `src/lib/lighthouse-summary.ts`.
- Local verification in this repo can create `assets/sveltekit-report-app-template/node_modules/`, but that directory is template-local and ignored by the template’s own `.gitignore`.

## Maintenance Implications

- Feature work is usually cross-cutting: update docs, workflow guidance, and template code together.
- Repo-wide scan results belong in durable notes because the Brain-generated top-level docs start generic and do not automatically infer the actual planner/template split.
